🔧 重复API调用修复测试

修复 /api/v1/questions/generate 接口被调用两次的问题

🚨 问题分析

问题现象： `/api/v1/questions/generate` 接口被调用了两次

// 第一次调用：CompletePage.tsx const questionResult = await questionService.generateQuestions(questionData); // 第二次调用：MockInterviewPage.tsx if (stateData.sessionId) { questionData = await questionService.getSessionQuestions(stateData.sessionId); } else { // 如果getSessionQuestions失败，会进入这个分支 questionData = await questionService.generateQuestions({...}); }

根本原因：

  • CompletePage 已经生成了问题并传递了数据
  • MockInterviewPage 没有充分利用已传递的问题数据
  • 当 getSessionQuestions 失败时会重新生成问题

✅ 解决方案

修复策略： 优先使用已传递的问题数据，避免重复API调用

// 修复后的逻辑 if (stateData.sessionId && stateData.questions) { // 直接使用已传递的问题数据，避免重复API调用 setQuestions(stateData.questions); // 只获取会话信息，不重新获取问题 sessionData = await questionService.getSessionQuestions(stateData.sessionId); setInterviewSession(sessionData.session); } else if (stateData.sessionId) { // 只有会话ID，需要获取问题 questionData = await questionService.getSessionQuestions(stateData.sessionId); setQuestions(questionData.questions); setInterviewSession(questionData.session); } else { // 只有在没有会话ID时才创建新的面试会话 // ... 创建新会话的逻辑 }
